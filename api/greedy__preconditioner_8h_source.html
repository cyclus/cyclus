00024 /// @class GreedyPreconditioner
00025 ///
00026 /// @brief A GreedyPreconditioner conditions an ExchangeGraph for a GreedySolver
00027 /// by ordering the RequestGroups and ExchangeNodes within each RequestGroup
00028 /// weighted by their commodity's importance. The Graph is conditioned in-place.
00029 ///
00030 /// @section weighting Weighting
00031 ///
00032 /// Commodity weights are provided to the conditioner via its constructor. A
00033 /// larger weight implies a higher level of importance for solving.
00034 ///
00035 /// The conditioning weight for a node is calculated as $w_cond_i = w_commod_i *
00036 /// (1 + \frac{\overline{p_i}}{1 + \overline{p_i}})$, where $w_cond_i$ is the
00037 /// calculated conditioning weight, $w_commod_i$ is node $i$'s commodity's
00038 /// weight, and $\overline{p_i}$ is the average preference of all bid arcs
00039 /// associated with node $i$.
00040 ///
00041 /// First, the ExchangeNodes of each RequestGroup are sorted according to
00042 /// their conditioning weights. Then, the average weight of each RequestGroup is
00043 /// determined. Finally, each RequestGroup is sorted according to their average
00044 /// weight.
00045 ///
00046 /// @section example Example
00047 /// Consider the following commodity-to-weight mapping: {"spam": 5, "eggs": 2}.
00048 /// Now consider two RequestGroups with the following commodities:
00049 ///   #. g1 = {"eggs", "spam", "eggs"}
00050 ///   #. g2 = {"eggs", "spam"}
00051 /// And the following preference-commodity  mapping:
00052 /// {g1: {"spam": 3/4, "eggs": 1/4}, g2: {"spam": 1, "eggs": 1}.
00053 ///
00054 /// First, the groups will be ordered by conditioning weights:
00055 ///   #. g1 = {"spam", "eggs", "eggs"}
00056 ///   #. g2 = {"spam", "eggs"}
00057 ///
00058 /// Finally, the groups themselves will be ordered by average weight:
00059 ///   #. {g2, g1}
