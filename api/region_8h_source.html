00018 /// @section intro Introduction
00019 /// The Region type assists in defining the region-institution-facility
00020 /// hierarchy in Cyclus. A Region region is an actor associated with a set
00021 /// of institutions or facilities for which it is responsible. A Region may
00022 /// be used to adjust preferences in the ResourceExchange to make material
00023 /// routing decisions based on interfacility relationships. Deployment is a
00024 /// primary differentiator between different Region implementations.
00025 ///
00026 /// Like all agent implementations, there are a number of implementations
00027 /// that are distributed as part of the core Cyclus application as well
00028 /// as implementations contributed by third-party developers. The links
00029 /// below describe additional parameters necessary for the complete
00030 /// definition of a region of that implementation.
00031 ///
00032 /// @section functionality Basic Functionality
00033 /// All regions perform three major functions:
00034 ///
00035 /// -# Schedule the deployment of facilities by either
00036 /// -# Determining when new facilities need to be built, or
00037 /// -# Deferring to an Institution to make this determination.
00038 /// -# Manage the deployment of facilities by interacting with the
00039 /// Institutions to select a specific facility type and facility
00040 /// parameters
00041 ///
00042 /// Different regional types will be required to fully define the first
00043 /// two functions while the third is a built-in capability for all region
00044 /// types. For instance, one may wish to include a region which has
00045 /// exponential growth as its driving factor for facility creation or one
00046 /// may wish to have pre-determined building order based on time step
00047 /// (e.g. the JAEA benchmark). Additionally, one may wish for there to be
00048 /// a one-to-one region-to-instituion deployment for simple agents and
00049 /// thus demand that each instiution simply build a facility when its
00050 /// region determines the facility's necessity. However, one may instead
00051 /// wish to have two competing instiutions in one region and have the
00052 /// institution which provides the best incentive to the region to build
00053 /// a required facility.
00054 /// @section availableCoreImpl Available Core Implementation
00055 /// - NullRegion: This region is associated with a set of allowed
00056 /// facilities. It defers to an institution regarding facility
00057 /// deployment, making no demands on facility type or parameter (save the
00058 /// facility's allowability in the region). It makes no alterations to
00059 /// messages passed through it in either the up or down direction.
